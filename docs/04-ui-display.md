# 04 · 介面顯示

## 主畫面 UI 元素清單

| 編號 | UI 元素 | 說明 | 字數/位元顯示上限 |
|------|---------|------|-------------------|
| 1.0 | 資產欄位 | 顯示玩家的金幣與寶石，左側金幣圖標+數值／右側寶石圖標+數值，常駐於畫面上方 | 金幣：15 位元（超過 12 位元縮小顯示）；寶石：15 位元（超過 12 位元縮小顯示） |
| 2.0 | 小瑪莉輪盤 | 橢圓形主遊戲轉盤，約 24 格，燈號循環跑動決定獎勵 | 各格子數字依押注額等比連動（K 為單位） |
| 3.0 | 押注區 | 「押注」標籤 + 金幣圖案 + 數值 + 藍色 +/- 按鈕 | 金幣圖案 + 9 位元 |
| 4.0 | 【開始/領取】按鈕 | 綠色按鈕；短按開始遊玩；長按約 2 秒觸發自動旋轉；結果確定後變為【領取】 | — |
| 5.0 | 每日剩餘次數 | 畫面下方顯示「💎 金 今日剩餘旋轉 X/Y 次」 | — |
| 6.0 | 寶石贏取提示 | 獲獎時右上角寶石欄位旁顯示「+X,000」浮動數字 | — |
| 7.0 | 規則 | 顯示玩家當前 VIP 的各等級可遊玩次數 | — |
| 8.0 | 【關閉】按鈕 | 按下後關閉介面 | — |

## 轉盤版面規格

| 項目 | 規格 |
|------|------|
| 形狀 | 橢圓形（水平方向較寬） |
| 格子總數 | 約 24 格（含上方猴爺 1 格、下方聚寶盆 1 格） |
| 上方特殊格 | 猴爺圖 + 金邊寶石數量框（50M 押注時顯示「10K」） |
| 下方特殊格 | 聚寶盆圖 + 金邊寶石數量框（50M 押注時顯示「8K」） |
| 高獎勵格 | 紫底六角形，尺寸較大（50M 時為 3K～6K），約 8 格散佈於轉盤四周 |
| 一般格（紅底） | 紅底圓形，中等獎勵（50M 時為 2K 等） |
| 一般格（綠/青底） | 綠底/青底圓形，一般獎勵（50M 時為 1K、1.5K） |

> 切換押注額時，所有格子的數字等比同步變化（如 50M→100M 時，全部 ×2），並根據位數縮放字體大小。

## 按鈕狀態對照

### 【開始/領取】按鈕

| 狀態 | 按鈕文案 | 副文字 | 按鈕行為 | 視覺表現 |
|------|----------|--------|----------|----------|
| 閒置可遊玩 | **開始** | 長按啟動自動旋轉 | 短按 → 開始遊玩；長按約 2 秒 → 跳出自動旋轉面板 | 綠色按鈕，可點擊 |
| 燈號跑動中 | — | — | 不可點擊 | 壓暗 |
| 結果確定（手動模式） | **領取** | — | 點擊 → 寶石從選中格子飛入資產欄位 | 綠色按鈕，可點擊 |
| 自動旋轉中 | **{剩餘手數}** | 停止 | 點擊 → 停止自動旋轉 | 綠色按鈕，上方大字顯示剩餘數字（如「10」「9」…），下方小字「停止」 |
| 次數歸零 | 冷卻文案 + 倒數 | — | 不可點擊 | 壓暗 + 顯示冷卻倒數 |

### +/- 押注切換按鈕

| 狀態 | 行為 | 視覺 |
|------|------|------|
| 正常 | 按 + 切到下一個押注額，按 - 切到前一個 | 藍色按鈕 |
| 到達最大額（500M）按 + | 循環跳至最小額（50M） | — |
| 到達最小額（50M）按 - | 循環跳至最大額（500M） | — |
| 燈號跑動中 | 不可點擊 | 壓暗 |
| 次數歸零 | 不可點擊 | 壓暗 |

## 自動旋轉選擇面板

| 項目 | 規格 |
|------|------|
| 觸發方式 | 長按【開始】按鈕約 2 秒 |
| 面板位置 | 覆蓋於轉盤中央區域 |
| 面板背景 | 半透明深色底 |
| 選項數量 | 4 個 |
| 選項內容（由上至下） | ∞ → 100 → 50 → 10 |
| 選項排列 | 垂直排列，每個選項為一條橫列 |
| 面板行為 | 不可滑動；點擊任一選項或外部位置皆會關閉面板 |
| 選擇後 | 按鈕文案變為「{手數} / 停止」；選擇 ∞ 則顯示「∞ / 停止」 |

## 每日剩餘次數顯示

| 項目 | 說明 |
|------|------|
| 顯示位置 | 畫面下方，按鈕下方的說明文字區域 |
| 格式 | 「💎 金 今日剩餘旋轉 X/Y 次」（如「今日剩餘旋轉 19/20 次」） |
| X 的含義 | 剩餘可用次數 |
| Y 的含義 | 每日總上限 |
| 行為 | 每次遊玩後 X 即時遞減 |

## 寶石贏取浮動提示

| 項目 | 說明 |
|------|------|
| 觸發時機 | 燈號停止確定獎項後 |
| 顯示位置 | 右上角寶石欄位下方 |
| 格式 | 「+X,000」（如「+2,000」「+1,000」） |
| 動畫 | 數字向上浮動後淡出 |
| 持續時間 | 約 1～2 秒（目測） |

## 金幣不足提示（⚠️ 建議規格）

| 項目 | 說明 |
|------|------|
| 觸發條件 | 玩家金幣餘額 < 當前押注額，點擊【開始】時觸發 |
| 提示類型 | 畫面中央彈出 Toast / 對話框 |
| 提示文案 | 「金幣不足，無法押注」（多語系見下方文案表） |
| 提示持續 | 約 2 秒後自動消失（若為 Toast）；若為對話框則需手動關閉 |
| 按鈕行為 | 【開始】按鈕不做動作，保持可點擊狀態 |
| 備註 | 具體提示樣式（Toast 或 Dialog）待 UI 設計確認 |
